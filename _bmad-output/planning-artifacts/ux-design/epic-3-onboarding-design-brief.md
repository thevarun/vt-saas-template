# Epic 3: User Onboarding & Welcome - Design Brief

## Overview

This design covers the complete user onboarding experience for new users who have just verified their email. The goal is to make new users feel guided and welcomed rather than dumped on an empty dashboard.

**Scope includes:**
- 3-step onboarding wizard (Username → Feature Tour → Preferences)
- Dashboard welcome state for first-time users
- Reusable EmptyState component for the design system
- Loading/Skeleton patterns for consistent loading states

## Design Decisions

### Onboarding Wizard
1. **3-step flow** - Short enough to complete, meaningful enough to personalize
2. **Progress indicator** - Visual dots/bar showing current step (1 of 3)
3. **Skip option** - Always available but not prominent (encourages completion)
4. **Centered card layout** - Consistent with auth pages, focused single-task UI
5. **Username in Step 1** - Establishes identity, reuses UsernameInput pattern from Epic-2

### Dashboard Welcome
1. **Personalized greeting** - Uses username from onboarding
2. **Action cards** - 2-3 clear next steps with CTAs
3. **Progressive disclosure** - Welcome state transitions to regular dashboard with activity

### Component Library
1. **EmptyState** - Reusable with icon, title, description, CTA props
2. **Skeleton loaders** - Match content shape (card, text, avatar variants)
3. **Spinner** - Multiple sizes (sm, md, lg) using Loader2 icon

## Visual Direction

- **Style:** Clean, modern aesthetic inspired by Linear/Notion
- **Layout:** Centered cards with subtle shadows on gradient backgrounds
- **Progress:** Visual step indicator with completed/active/pending states
- **Spacing:** Consistent padding (p-6 to p-10), clear visual hierarchy
- **Animation:** Subtle transitions, pulse/shimmer for loading states
- **Responsiveness:** Mobile-first, cards stack vertically on small screens
